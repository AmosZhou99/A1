Project Ideas

Overview: My project idea is to make a simple game using Unity. Its name is 2D-Ball based on a very famous game called 3D Pin-Ball. Due to the fact that I used to play this game through WeChat microprogram, it is very fun and seems not that hard for starters to make it come true. Therefore, I decided to make this game for my following assignment.

Motivation: Mainly because I am a starter for IT industry, it will be hard to realize the complicated program. Simple game like 2D-Ball is not only a challenge for starters but a good chance to learn computer graphics concepts and put them into practice. The most of all, it is doable and I have clues on where to start especially I have learned some basic concepts in computer graphics by OpenGL.

Description: After seeking for information online, making a 2D game basically need two steps. The first step is to build the scene using Unity or other graphic tools. As a start of the scene, it is better for us to build boundaries like walls to limit the moving space for balls. Choosing special material textures to make sure ball can bounce back from the wall. Then, make the ways for the ball to make sure the ball can bounce following physical rules. Next, set up a muzzle for the ball to define the start point. Add a text object in order to record the scores which counts the bouncing times of the ball. Then, put a level panel into the scene mainly to divide the scene into a lot of grids which will be filled with obstacles randomly. The whole structure of game will be done at that point. However, we still need a camera to fix the scene in 2D. After that it is time to decorate the scene, creating certain simple geometric figures to be the obstacles and adding some stunts using certain letters to stand for its function. For example, we can add a number into the geometric figure to stand for its remaining times to be bounced or add a letter ‘B’ for Big to make the ball bigger for some time. After all the procedures, I think we finish building the scene. It comes to the next step, writing codes to make the scene work as a game. First, I believe plenty of generators are necessary to generate all the obstacles and stunts. Besides, we need a state monitor to record what state we are in such as achieving game goal or already game over. Next, we should create a ball state monitor as well to record which state the ball is in, for example if the ball gets any stunts. For the ball, we still need an interactive system to make it work when the ball meets any obstacles and when it happens, we should decrease number of the obstacle. Further, we need a detector to see if any ball reaches the bottom of the scene and set an action to make the ball return to the muzzle so that player can launch the ball again. After that, an aiming system is essential in order that we can only use mouse to let the ball launch at the right angle towards bottom. In order to monitor all the status of obstacles, we can set a monitor on each obstacle to record the status. If obstacle’s number counts back to zero, it will vanish automatically. Setting an action on all the stunts which will deliver a message to change the state of the ball, which may make the ball bigger or make a clone of the ball itself. Finally, we just need one more recorder for the state of the game itself to pop up the main menu when game is over or paused. Combining two steps together, I think this game will work.

Tools and Technologies: Unity is the main tool for this project, it is a powerful cross-platform game engine. Plus, it needs to be coded by some basic coding languages such as C++ and C#.

Skills Required: C# language is mainly needed for coding in Unity. Although I only know some basic concepts of C#, there are plenty of tutoring videos and I can use the resources of library teaching how to use Unity to develop a simple game.
